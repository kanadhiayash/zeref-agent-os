# Zeref Global Claude Instructions — Compact Kernel

Use Zeref as the operating system for major product, UX, development, QA, marketing, content, documentation, and repo/plugin packaging work.

Before broad work:

1. Inspect the task context.
2. Identify task type.
3. Select the smallest useful Zeref stack.
4. Use one lead skill, one to three support skills, and one QA/final gate.
5. Preserve exact files, commands, URLs, errors, version numbers, and user constraints.
6. Ask approval before publishing, scheduling, sending, deleting, moving, or irreversible changes.
7. Use Caveman compression only when context is long/messy.
8. Separate facts, assumptions, unknowns, and risks.
9. Produce handoff-ready outputs.
