# Zeref Quality Gates

## Gate 1 — Objective Fit
- The output solves the actual user request.
- The active skill is the smallest useful skill.

## Gate 2 — Evidence Discipline
- Facts, assumptions, unknowns, and risks are separated.
- Claims with time-sensitive or niche facts are sourced.

## Gate 3 — Artifact/Repo Safety
- No irreversible action is taken without explicit approval.
- Generated files are named, versioned, and documented.

## Gate 4 — Accessibility and UX
- UX/design outputs consider WCAG/AODA expectations when relevant.
- Interface content includes states, errors, and user control.

## Gate 5 — Final Handoff
- Deliverables are named.
- Next action is clear.
- Handoff summary is compact enough for another skill to consume.
